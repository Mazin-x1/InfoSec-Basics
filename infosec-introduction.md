# Information Security (InfoSec) - Introduction

## Definition
Information Security (InfoSec) is the practice of protecting data and systems from unauthorized access, modification, or disruption.

---

## Infrastructure Basics

### Client
A user device (PC or laptop) used to access services and resources.

### Server
A machine that provides specific services such as web hosting or databases.

### Cloud
A collection of interconnected servers in data centers that provide scalable services.

### Network
A group of connected devices that can communicate with each other.

---

## Core Concept (CIA Triad)

### Confidentiality
Ensuring that only authorized individuals can access specific information.

### Integrity
Ensuring that data is not altered without authorization.

### Availability
Ensuring that systems and data are accessible when needed.

---

## Risk, Threat, Vulnerability

### Threat
Any potential cause of harm (e.g., hacker, malware, natural disaster).

### Vulnerability
A weakness in a system that can be exploited (e.g., bug, weak password).

### Risk
The likelihood of a threat exploiting a vulnerability and causing damage.

---

## Why Important
- Protect sensitive data
- Prevent cyber attacks
- Maintain system reliability

---

## Where Used
- Networks
- Cloud systems
- Applications
- Databases

---

## Simple Scenario
A company stores customer data:

- If data is leaked → Confidentiality is broken
- If data is modified → Integrity is broken
- If system goes down → Availability is broken

---

## Industry Roles (Overview)

- CISO: Oversees security strategy
- Architect: Designs secure systems
- Penetration Tester: Simulates attacks
- Analyst: Monitors threats

---
